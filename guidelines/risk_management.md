# 리스크 관리 정책

## 3단계 리스크 관리

### 1단계: 에이전트 레벨
각 에이전트는 독립적인 리스크 한도를 가짐:
- **포지션 한도**: 배분받은 자본의 40% 이내
- **주문 쿨다운**: 60초 (동일 에이전트 연속 주문 방지)
- **최소 주문 금액**: 5,000 KRW (빗썸 최소 주문)
- **잔고 확인**: 주문 전 KRW/자산 잔고 확인

### 2단계: 포트폴리오 레벨
전체 포트폴리오에 대한 통합 리스크 관리:
- **일일 손실 한도**: 30,000 KRW (시작 자본의 10%)
- **총 손실 한도**: 90,000 KRW (시작 자본의 30%)
- **최대 포지션 비율**: 전체 자산의 80% 이내 (최소 20% 현금 보유)
- **일일 P&L 추적**: 모든 실현 손익을 일자별로 집계

### 3단계: 서킷브레이커
비상 정지 조건 (하나라도 충족 시 전체 거래 중단):
1. 일일 손실 > 30,000 KRW
2. 총 손실 > 90,000 KRW
3. 연속 손실 5회 이상
4. `data/KILL` 파일 존재

## 리스크 지표 모니터링

### Claude Code가 확인해야 할 항목
매 리포트 확인 시:
- 일일 P&L이 -20,000 KRW 이하면 주의 (경고 단계)
- 총 P&L이 -60,000 KRW 이하면 거래 축소 권장
- 연속 손실 3회 이상이면 에이전트 점검 필요
- 특정 에이전트 MDD 20% 초과 시 벤치 고려

### 시장 상황별 리스크 조절
- **고변동성**: Temperature 높이기 (3.0+), 주문 크기 축소
- **저변동성/횡보**: Temperature 낮추기 (1.5), 평균회귀 에이전트 우대
- **급등/급락**: 즉시 거래 중단, 시장 안정화 후 재개

## 복구 절차
1. 서킷브레이커 발동 → `data/circuit_breaker.json` 확인
2. 원인 분석 (어떤 에이전트가 손실 유발?)
3. 필요 시 에이전트 벤치/은퇴 처리
4. 설정 조정 (한도, 쿨다운 등)
5. `data/KILL` 파일 삭제 후 재시작
